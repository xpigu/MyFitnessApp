# 饮食模块升级计划

## 1. 当前问题

1. 当前“添加食物”使用随机卡路里，缺少真实食物与份量模型。
2. 卡路里只记录结果，不体现计算过程，用户难以理解数据来源。
3. 饮食建议只有静态文案，没有根据当天摄入、剩余预算和三餐完成度动态规划。
4. 缺少“健康食谱规划”功能，无法把记录行为转化为行动建议。

## 2. 本次改造目标

1. 建立基础食物库，支持用户按餐次选择常见食物。
2. 用“每 100g 热量 x 实际克数 / 100”的方式计算摄入卡路里。
3. 保留卡路里结果，同时展示份量和计算依据，增强可解释性。
4. 根据每日热量预算、已摄入热量、运动消耗和已完成餐次，生成健康食谱规划。

## 3. 设计方案

### 3.1 食物选择与热量算法

- 食物数据来源：内置基础食物库。
- 记录维度：餐次、食物名称、份量、对应热量、备注。
- 计算公式：`calories = round(caloriesPer100g * grams / 100)`.
- 交互流程：
  1. 用户点击“添加食物”
  2. 选择当前餐次可用食物
  3. 选择份量
  4. 系统自动计算卡路里并写入记录

### 3.2 健康食谱规划

- 输入：目标热量、已摄入热量、运动消耗、已完成餐次、当前时间。
- 输出：下一餐优先建议 + 今日完整食谱建议。
- 规划逻辑：
  1. 计算剩余热量
  2. 根据早餐/午餐/晚餐/加餐比例分配目标热量
  3. 根据当前时间和缺失餐次判断“下一餐”
  4. 从模板中选择更轻、更标准或更高蛋白的搭配

## 4. 分阶段实施

### 第一阶段：已完成

1. 新增基础食物库和卡路里计算模型。
2. 将随机添加食物替换为“选食物 + 选份量 + 自动计算卡路里”。
3. 将“快速加餐”升级为“智能加餐”。

### 第二阶段：已完成初版

1. 在饮食卡片中加入“健康食谱规划”入口。
2. 根据当天剩余热量和已完成餐次生成推荐摘要。
3. 支持弹窗查看今日完整食谱规划。

### 第三阶段：后续建议

1. 支持自定义食物、收藏常用搭配和搜索。
2. 将食谱规划沉淀成独立页面，并支持批量编辑和替换推荐项。
3. 结合用户身体数据与阶段训练计划，动态调整饮食目标热量。
4. 增加历史营养素趋势和目标达成分析。

### 第四阶段：已完成初版

1. 新增减脂、维持、增肌三类饮食目标。
2. 支持在首页切换当前饮食目标，并持久化保存。
3. 食谱规划会按目标动态调整热量分配、模板偏好和营养素建议。
4. 营养素概览已显示“今日摄入 / 当前目标 / 下一餐推荐”三层信息。

### 第五阶段：已完成初版

1. 新增自定义食物存储能力，支持录入每100g热量与三大营养素。
2. 自定义食物已接入餐次选择器，可与内置食物一起添加。
3. 支持将当前推荐下一餐收藏为常用组合。
4. 支持从常用组合中直接一键套用到当天饮食记录。

### 第六阶段：已完成初版

1. 在饮食卡片中新增“饮食洞察”，展示目标达成分析与近7天趋势。
2. 可根据当前饮食目标计算蛋白质、碳水、脂肪的今日达成率。
3. 可根据近7天饮食记录生成平均营养素摄入趋势。
4. 系统会比较“今天”和“近7天平均”的贴近程度，给出趋势信号。
